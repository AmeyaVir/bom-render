Spud Activity Prediction for Mineral Asset Valuation

Advanced Machine Learning Framework for Strategic Decision-Making

Executive Summary

Stone Hill Minerals requires a quantitative model to predict spud events on its undeveloped acreage sections within the next 6 to 18 months. This prediction must directly support a "Keep vs. Sell" strategic decision by assigning a probabilistic economic value to the undeveloped asset.

Primary Project Goal

To integrate Machine Learning predictions of operator drilling activity into a Real Options Valuation (ROV) framework to determine the optimal hold period or exit price for undeveloped mineral and royalty assets.

Project Duration

24 weeks (6 months)

Success Metric

ROV-based valuation accuracy >85%

ML Classification

Spud probability prediction using advanced feature engineering

Real Options Valuation

Financial modeling with flexibility premium calculation

Strategic Decision

Keep vs. Sell framework based on quantitative analysis

Critical Business Scoping Questions

These questions focus the project on maximizing Return on Investment (ROI) and minimizing strategic risk.

ROI & Performance Questions
  • **Monetization Impact:** How will the model's **Precision@K (Top 10%)** metric directly translate into realized value (e.g., dollars saved from avoiding a premature sell, or dollars gained from a timely spud)? Is the **85% decision accuracy** target financially measurable?
  • **Acquisition Thresholds:** How can the resulting ROV/EMV value be used to set the **Max Bid Price** for undeveloped acreage acquisitions, ensuring SHM only purchases assets where the $\text{V}_{\text{ROV}}$ provides a sufficient margin over the asking price?
  • **Resource Allocation:** If the model identifies a high-$p$ asset (high spud probability) but the operator is financially weak, should SHM commit resources to encouraging the spud, or is the prediction itself the final input for the **Keep vs. Sell** decision?
Risk & Decision Logic Questions
  • **Threshold Strategy:** What is the specific business-approved **risk tolerance** that defines the "Sell" zone? For example, does a low spud probability ($p < 30\%$) coupled with a short **T-T-E** ($< 6$ months) automatically mandate a "Sell" recommendation, even if the **EMV** is slightly above the market price?
  • **Explainability & Trust:** Since the ML model's output ($p$) feeds the financial model (ROV/DTA), how will the **Feature Importance** be presented to non-technical stakeholders (e.g., leadership, finance team) to ensure they **trust** the $p$ value and adopt the resulting **Keep vs. Sell** recommendation?
  • **Data Cost Justification:** Given the cost of high-value data (e.g., DrillingInfo/Enverus $$$), what is the calculated **marginal financial value** that each external data source provides to the final decision accuracy, and should any expensive, low-impact data source be descope'd?

Valuation Framework

Real Options Valuation and Decision Tree Analysis methodologies

Real Options Valuation (ROV) Framework

Since Stone Hill's asset is an undeveloped mineral right (an American-style option to drill, held by the operator), a static NPV/DCF is insufficient. The valuation must capture the flexibility value inherent in the asset.

Binomial/Trinomial Lattice Model

ROV calculates the full value of the undeveloped asset as:

Asset Value = Intrinsic Value + Time Value/Flexibility Premium
ROV Input ML/Data Source Mathematical Representation
Spud Probability (p) ML Classification Output State transition probability
Time to Expiration (T) Lease Contract Data Option life parameter
Volatility (σ) Commodity Markets Price volatility input
Exercise Price (K) Drilling Cost Estimates Development cost threshold
Underlying Value (S) Expected EUR × Prices Asset intrinsic value
Decision Node Logic

At each time step, the operator chooses:

  • Exercise: Drill the well
  • Hold: Wait for better conditions
  • Abandon: Let lease expire
Key Advantages:
  • Captures timing optionality
  • Incorporates uncertainty
  • Quantifies flexibility value
  • Supports sensitivity analysis
Decision Tree Analysis (DTA)
Expected Monetary Value (EMV) Calculation

A simpler, more transparent method for direct strategic decision-making:

EMVKeep = (p × NPVRoyalty) + ((1-p) × ValueLapsed Lease)
Decision Rules:
  • KEEP: If EMVKeep > Current Market Sell Price
  • SELL: If EMVKeep < Current Market Sell Price
  • HOLD: If difference is within uncertainty range
Scenario Analysis Framework
Scenario Spud Prob. Action Rationale
High Prob + Long T-T-E >60% KEEP High upside, low risk
High Prob + Short T-T-E >60% KEEP Immediate cash flow
Low Prob + Long T-T-E <30% HOLD Wait for better intel
Low Prob + Short T-T-E <30% SELL Avoid total loss
Integrated "Keep vs. Sell" Framework
Final Decision Logic

The ultimate decision rests on comparing the intrinsic and option values against the market value:

Keep Decision ⟺ VROV (Intrinsic + Option) > VMarket (Comparable Sales)
Factors Favoring "KEEP":
  • High spud probability (>50%)
  • Strong geological quality
  • Active operator in area
  • Recent permit activity
  • High commodity prices
  • Low market valuations
  • Long time to expiration
Factors Favoring "SELL":
  • Low spud probability (<30%)
  • Marginal geological quality
  • Inactive operator
  • No recent drilling activity
  • Depressed commodity prices
  • High market valuations
  • Short time to expiration
Strategic Insights

A high Spud Probability Score (p) combined with a short T-T-E should increase the VROV sharply, pushing the asset toward the "Keep" category for immediate royalty income.

Conversely, a low p with a short T-T-E signals high risk and an imminent loss of the asset, favoring a Sell before the lease lapses.

Glossary & Technical Reference

Comprehensive definitions and contextual explanations

Oil & Gas Terms
Spud
The act of beginning to drill a well. Spud date marks the official start of drilling operations.
EUR (Estimated Ultimate Recovery)
The total amount of oil or gas expected to be produced from a well over its entire productive life.
HBP (Held by Production)
A lease status where production from existing wells maintains the lease beyond its primary term.
Mineral Interest
Ownership of the minerals beneath the surface, including the right to lease and receive royalty payments.
Royalty Interest
A percentage of production revenue paid to the mineral owner, free of drilling and operating costs.
T-T-E (Time to Expiration)
The remaining time before a lease expires, critical for drilling timing decisions.
WIP (Well in Progress)
A well that has been spudded but not yet completed and brought into production.
Type Curve
A representative production decline curve used to estimate future well performance.
Lateral Length
The horizontal length of a wellbore, typically measured in feet, affecting total production.
Net Pay
The thickness of rock formation that contains economically producible hydrocarbons.
Financial & Valuation Terms
Real Options Valuation (ROV)
A valuation method that captures the value of flexibility in timing investment decisions under uncertainty.
Decision Tree Analysis (DTA)
A decision-making tool that maps out alternatives and their probable outcomes in a tree-like format.
Expected Monetary Value (EMV)
The probability-weighted average of all possible outcomes in a decision scenario.
Intrinsic Value
The immediate exercise value of an option, calculated as the difference between asset value and exercise cost.
Time Value
The portion of an option's value attributable to the time remaining until expiration.
Volatility (σ)
A measure of price fluctuation, typically expressed as an annualized standard deviation.
Exercise Price (K)
The cost of development (drilling and completion) required to extract value from the asset.
Binomial Lattice
A numerical method for valuing options by modeling price movements as a series of up/down steps.
Monte Carlo Simulation
A computational method using random sampling to model complex systems with multiple uncertainties.
Net Present Value (NPV)
The present value of future cash flows minus the initial investment, discounted at an appropriate rate.
Machine Learning Terms
AUC-ROC
Area Under the Curve - Receiver Operating Characteristic. Measures a classifier's ability to distinguish between classes.
Feature Engineering
The process of creating new input variables from raw data to improve model performance.
Hyperparameter Tuning
The process of optimizing model parameters that control the learning process.
Cross-Validation
A technique for assessing model performance by partitioning data and testing on unseen subsets.
Precision@K
The percentage of correctly predicted positive cases among the top K predictions.
Model Calibration
The degree to which predicted probabilities match actual frequencies of positive outcomes.
Feature Importance
A measure of how much each input variable contributes to the model's predictions.
Overfitting
When a model learns training data too specifically and fails to generalize to new data.
Ensemble Methods
Techniques that combine multiple models to create a stronger predictor than any individual model.
MLflow
An open-source platform for managing the machine learning lifecycle, including experiments and models.
Technical & Data Terms
Databricks
A unified analytics platform for big data processing and machine learning, built on Apache Spark.
Feature Store
A centralized repository for storing, managing, and serving machine learning features across teams.
Delta Lake
An open-source storage layer that brings ACID transactions to Apache Spark and big data workloads.
ETL Pipeline
Extract, Transform, Load - the process of moving and transforming data from sources to destinations.
API (Application Programming Interface)
A set of protocols and tools for building software applications and enabling system integration.
Data Lineage
The tracking of data from its origin through all transformations to its final destination.
Schema Drift
Changes in data structure over time that can break downstream processing and analytics.
SCADA
Supervisory Control and Data Acquisition - systems for monitoring and controlling industrial processes.
GIS (Geographic Information System)
Technology for analyzing and visualizing spatial and geographic data.
REST API
Representational State Transfer - an architectural style for web services enabling data exchange.
